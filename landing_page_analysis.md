# 소노아임레디 랜딩페이지 (tossads-sono-landing) 구현 분석서

이 문서는 다음 작업자(AI 에이전트 또는 개발자)가 기존 컨텍스트를 빠르게 이해하고 일관성 있게 작업을 이어가기 위해 작성된 `index.html` 및 `style.css` 구조 분석 가이드입니다.

---

## 1. 프로젝트 개요 (Core Stack)
- **접근 방식**: 프레임워크 없는 **Vanilla HTML / CSS / JavaScript** 구조.
- **애니메이션 코어**: **GSAP (GreenSock)** 및 **ScrollTrigger** 플러그인, 텍스트 효과를 위한 **Splitting.js**.
- **목적**: 매끄럽고 극적인 스크롤 애니메이션과 높은 퀄리티의 UI 요소를 활용하여 '소노아임레디' 가전 렌탈 및 상조 결합 서비스를 홍보하고 상담 신청을 유도.

## 2. 디자인 시스템 및 테마 (Design System)
절대적으로 유지해야 할 디자인 룰입니다.

- **기본 컬러 테마**: 철저한 **Black & White** (백그라운드는 완전한 `#fff` 또는 완전한 블랙/다크 테마 `#000`, `121212`, `#1d1d1f`). 배경색 혼용(`.f8f9fb` 등)은 통일감을 해치므로 화이트로 통일됨.
- **포인트 컬러**:
  - 브랜드 네이비: `#001A3D` (주로 굵은 타이포그래피에 사용)
  - 다크 허니 골드(Accent): `#C5A065` (강조할 숫자, 뱃지, 부가 버튼 등에 사용)
  - 스마트케어 브랜드 핑크(Action): `#ff1f5d` (가전 선택 탭, 중요 환급 금액, 계산기 마이너스(-) 할인 시각화, 메인 CTA 버튼 등에 사용하여 즉각적 전환 유도)
- **타이포그래피**: `Pretendard` 등 크로스 플랫폼 산세리프 폰트 스택. 다양한 사이즈를 CSS 변수(`--font-hero`, `--font-h1` 등)로 관리. 특수하게 화면 가로폭 기반의 단위(`vw`)를 적극 사용하여 브라우저 크기에 비례하는 극적인 텍스트 스케일링을 구현함.
- **UI 특징**: 둥근 모서리(`border-radius: 20px~30px`), 부드럽고 넓은 그림자(`box-shadow`) 적용, 글래스모피즘(`backdrop-filter: blur()`).

## 3. 핵심 구조 및 섹션 기믹 (Section Architecture)

### A. 히어로 섹션 (Hero Section) & 3D 날씨 아이콘
- 스크롤에 따라 투명도와 위치가 변하는 시네마틱 헤더.
- **Glassmorphism 아이콘**: `backdrop-filter: blur`를 활용한 3D 구름, 번개, 비, 무지개 아이콘 구현.
- **[Safari 크리티컬 이슈]**: iOS 사파리에서 `backdrop-filter`와 SVG `<clipPath>`를 혼용할 때 아이콘 프레임이 사각형으로 잘리거나(`display: none` 버그) 렌더링이 깨지는 고질적 이슈가 있음. **반드시 `-webkit-mask-image` 표기법과 `translateZ(0)`를 결합하여 해결된 방식을 유지**해야 함.

### B. 후킹 패널 (Hooking Panels - 가로/상하 스크롤 전환)
- GSAP `ScrollTrigger`를 이용해 화면을 핀(pin)으로 고정한 상태에서 내부 패널이 한 장씩 겹치면서(z-index 및 y축 이동) 드러나는 구조.
- 다크 모드, 라이트 모드가 전환되며 극적인 대비를 이룸.

### C. 스마트케어 요금제 및 가전 선택 (Smartcare Product & Calc UI)
- **목적**: 요금제(구좌수)별 상세 혜택과 체감 할인가를 유저가 직접 확인하게 하는 핵심 컨버전(Conversion) 영역.
- **주요 인터랙션 (GSAP + Vanilla JS)**:
  - **탭(Tab) 전환 애니메이션**: 탭 클릭 시 캐릭터 일러스트는 `elastic.out`, 해시태그 뱃지들은 엇박자로 `back.out`하며 통통 튀어나오게(stagger) 연출.
  - **가전 스와이프 캐러셀**: `requestAnimationFrame`을 이용해 가전 리스트가 항상 부드럽게 자동 스크롤(Ping-Pong)되도록 하여, 작은 모바일 화면(`42vw` 사이즈)에서도 직관적인 스와이프를 유도. 사용자 터치 시 자동 롤링 일시정지.
  - **영수증(계산기) 시각화**: 가전 제품 클릭 시 숨겨져 있던 영수증 UI가 `gsap.fromTo` 모션으로 등장하며, 기본 요금 - 제휴 할인 = 최종 체감가가 영수증처럼 출력되도록 하여 즉각적 구매 욕구 자극.

### D. 솔루션 & 하이브리드 인포 박스 (Hybrid Info)
- 플렉스박스 및 그리드 레이아웃의 조합. 
- 카드 위에 뱃지(`position: absolute`) 및 거대한 숫자 투명도 오버레이를 통해 시각적 깊이를 만듦.

### D. 프리미엄 카드 & 아코디언 명세 (Premium Cards)
- 사용자 상호작용(카드 클릭) 시 CSS가 아닌 **GSAP**를 활용하여 카드의 높이(`height`)를 부드럽게 펼쳐주는 아코디언 로직 구현.
- 클릭 시 펼쳐지는 내용으로 자동 `scrollTo` 되도록 설계.

### E. 무한 롤링 고객 리뷰 (Review Carousel)
- GSAP의 `.to()` 트윈에 `repeat: -1`과 `ease: "none"`을 부여하여 끊김 없이 흘러가는 마키(Marquee) 애니메이션 구현. (요소 복제를 통해 끊기지 않는 루프 생성).

### F. 폼 및 푸터 (Consultation & Footer)
- 입력 폼 영역은 화이트 톤으로 깔끔하게 유지하며 호버 시 약간 떠오르는(Y축 이동 및 박스섀도우 증가) 인터랙션을 가짐.
- 푸터(Footer) 상단부는 하단에서 올라오는 커대한 타이포그래피와 결합되어 있음.

### G. 대형 타이포그래피 등장 효과 (Final Typography Animation - *Crucial*)
- `Splitting.js`로 글자를 자음/단어 기준으로 쪼개어 span으로 감싼 뒤 제어.
- **SONO IMREADY**: 스크롤 시 화면 스크린 바깥(양쪽 끝)에서 중앙으로 자석처럼 수렴하며 합체되는 고도의 어트랙트 모션. `x` 오프셋을 현재 브라우저 넓이 비율 기반(ex: `window.innerWidth * 0.8`)으로 계산하여 극적인 다이내믹함을 유지함.

## 4. 후속 작업자(AI 에이전트)를 위한 가이드라인

1. **배경색 및 선 추가 시 주의사항**: 
   - 회색조(`#f8f9fb` 등) 사용 금지. 기본 배경은 무조건 흰색(`#fff`) 또는 상황에 따른 칠흑(`#000`,(`#1d1d1f`))을 유지하세요.
2. **사파리(Safari) 호환성 주의 지시**: 
   - Mask 영역 처리 시 SVG `clip-path`를 순수 CSS에 적용하지 마십시오. 백엔드가 깨집니다. 
   - 3D 효과나 `backdrop-filter` 추가 시 반드시 렌더링을 잡는 `-webkit-transform: translateZ(0)` 하드웨어 액셀러레이션을 세트로 제공하세요.
3. **GSAP 기반 이벤트 통일**:
   - 아코디언 또는 스크롤 추적 관련 스크립트를 수리할 때는 새 라이브러리 추가나 기본 CSS Transition 전환보다는 `index.js` 단의 GSAP 로직 내부에 `gsap.to()` 와 `ScrollTrigger`를 추가/수정하여 타이밍 충돌을 방지하세요.
4. **캐싱 이슈 해결**: 
   - `index.html` 상의 CSS 호출 버전을 항시 올려주세요. (`style.css?v=5.x`) 새로고침/캐시 문제인지 파악하는 것이 중요합니다.
5. **DOM 조작 구조**:
   - `index.html` 내부에 `<style>` 태그(인라인 블록)가 남아 있을 수 있으니 외부 `style.css` 수정을 할 때 HTML 내부에 덮어쓰기된 속성 여부를 교차 검색(grep)하세요. (특히 푸터 배경색 등의 버그 원인).
